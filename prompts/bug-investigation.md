# Bug 排查 Prompt

## 使用场景

用于根据错误日志、复现步骤、失败测试和相关代码定位后端问题。适合 Go、C#、Rust 项目中的接口异常、数据不一致、空指针、错误码异常、测试失败、性能突增等问题的初步分析。

## 输入字段

- 技术栈：语言、框架、测试框架、数据库或中间件。
- 问题现象：用户可见表现或系统异常表现。
- 复现步骤：触发问题的具体操作、请求参数、环境条件。
- 预期行为：系统本应返回或产生的结果。
- 实际行为：错误响应、异常日志、错误码、指标变化。
- 错误日志：关键日志、堆栈、请求 ID、时间范围。
- 相关代码：入口、核心逻辑、依赖接口、错误处理代码。
- 最近变更：近期合并、配置变更、依赖升级。
- 已排除项：人工已经验证过不是原因的方向。

技术栈：
[填写 Go/C#/Rust、框架、数据库、中间件、测试框架]

[填写用户现象或系统表现]

[填写可重复执行的步骤、请求、环境条件]

## 质量检查点

- 根因必须绑定日志、代码、复现步骤或测试失败证据。
- 必须区分事实、推测和需要验证的信息。
- 修复方案应优先选择最小改动，不引入无关重构。
- 必须包含能复现问题或覆盖风险的回归测试建议。
- 不得建议忽略错误、吞掉异常或绕过鉴权、校验、事务保护。
