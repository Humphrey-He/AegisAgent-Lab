# 需求拆解 Prompt

## 使用场景

用于把一个后端需求拆成 AI 可执行的小任务，并明确范围、风险、验收标准和测试计划。适合在正式编码前使用，尤其适合 Go、C#、Rust 项目的增量功能开发、接口补齐、模块改造和测试补齐。

## 输入字段

- 需求背景：业务来源、问题现状、用户价值。
- 目标行为：完成后系统应具备的可观察行为。
- 技术栈：语言、框架、存储、中间件、测试框架。
- 当前实现：相关模块、入口、接口、数据模型、相似实现。
- 约束条件：兼容性、安全、性能、错误码、接口契约、文件范围。
- 验收标准：测试、接口验证、日志、指标、人工检查。
- 非目标：本次明确不做的事项。

## 模板

```text
你是一个资深后端开发助手。请把以下需求拆解为适合 AI Coding 小步执行的任务。请先澄清需求边界，再给出实现顺序和验收方式。

需求背景：
[填写为什么要做、当前问题、业务价值]

目标行为：
[填写完成后系统应该如何表现]

技术栈：
[填写 Go/C#/Rust、框架、数据库、中间件、测试框架]

当前实现：
[粘贴相关目录、接口、核心代码、相似实现]

约束条件：
[填写不能改变的行为、兼容性要求、性能要求、安全要求、文件范围]

验收标准：
[填写测试命令、接口验证、日志或人工验收方式]

非目标：
[填写本次明确不处理的事项]
```

## 质量检查点

- 拆解后的每个任务都应能在单轮或少量轮次中完成并验证。
- 必须明确非目标，避免 AI 扩大范围。
- 必须给出受影响模块和测试计划。
- 必须指出需要人工确认的业务规则和高风险点。
- 不得直接跳到代码实现，先完成范围和验收定义。
