# 代码阅读 Prompt

## 使用场景

用于让 AI 阅读后端模块、调用链、接口契约或关键类型，帮助开发者快速理解 Go、C#、Rust 项目的现有实现。适合在需求拆解、Bug 排查、测试补齐、Review 前使用。

## 输入字段

- 阅读目标：需要理解的问题或即将修改的功能点。
- 技术栈：语言、框架、测试框架。
- 目录结构：相关模块和文件路径。
- 代码片段：入口、核心逻辑、类型定义、依赖接口。
- 相似实现：项目中可参考的同类代码。
- 关注点：调用链、错误处理、并发、事务、性能、安全、测试。
- 修改约束：不能改变的接口、行为或文件范围。

## 模板

```text
请作为后端代码阅读助手，阅读以下代码和目录信息。请先解释现有行为，不要直接给出改动代码。

阅读目标：
[说明要理解的问题，例如“订单创建链路如何校验库存”]

技术栈：
[填写 Go/C#/Rust、框架、测试框架]

目录结构：
[粘贴相关目录和文件路径]

关键代码：
[粘贴入口、核心逻辑、类型定义、依赖接口]

相似实现：
[粘贴同类模块或测试示例]

关注点：
[填写调用链、错误处理、事务、并发、安全、性能、测试等]

修改约束：
[填写不能改变的接口、行为、数据结构或文件范围]
```

## 输出格式

```text
1. 模块职责
   - 当前模块解决什么问题，边界是什么。

2. 关键类型与函数
   - 名称：
   - 职责：
   - 输入输出：
   - 重要副作用：

3. 主调用链
   - 从入口到依赖的步骤列表。

4. 外部依赖
   - 数据库、缓存、消息、HTTP/RPC、配置、时间、随机数等。

5. 错误处理与边界
   - 已处理的错误：
   - 可能遗漏的错误：

6. 测试现状
   - 已有覆盖：
   - 缺口：

7. 后续修改建议
   - 最小改动点：
   - 风险点：
   - 建议验证方式：
```

## 质量检查点

- 必须先描述现有行为，再提出修改建议。
- 必须指出信息不足处，不能把猜测当作事实。
- 必须识别入口、核心逻辑、依赖接口和测试位置。
- 修改建议应遵守给定约束和项目既有分层。
- 风险点要具体到行为、数据、错误处理或兼容性影响。
