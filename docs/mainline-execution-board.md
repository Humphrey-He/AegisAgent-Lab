# 主线一与主线二执行看板

## 目标

同时推进 AI Coding 工作流和 Agent 基础能力，形成可持续迭代的项目节奏。

## 当前并行范围

### 主线一：AI Coding 工作流

负责人范围：

- AI Coding SOP
- Prompt 模板
- 评估案例
- 基线报告模板
- AI Coding 实践报告模板

阶段验收：

- 每个 Prompt 能直接复用
- 每个评估案例能直接执行
- 每次实验能记录效率、质量和风险

### 主线二：Agent 基础能力

负责人范围：

- Go runtime
- C# API
- Rust tools
- Agent 架构文档
- 风险与 guardrails 文档

阶段验收：

- Go runtime 支持只读工具和 trace
- C# API 支持任务基础模型
- Rust tools 支持最小解析能力
- 三套工具链验证通过

## 每周节奏

### 周一：任务选择

- 从 `eval-cases/` 选择 1-2 个任务。
- 明确使用语言、目标项目和验收标准。
- 将任务记录到当周报告。

### 周二至周四：执行实验

- 主线一记录 AI Coding 的 Prompt 轮次、耗时、返工和测试结果。
- 主线二根据实验暴露的问题补充 Agent 工具能力。
- 所有高风险动作保持人工确认。

### 周五：复盘沉淀

- 更新 Prompt 模板。
- 更新评估案例。
- 更新 Agent 工具清单。
- 输出本周结论和下周任务。

## 决策规则

- 如果一个任务可以通过 Prompt 和人工执行解决，优先沉淀到主线一。
- 如果一个任务重复出现且需要工具调用，升级到主线二。
- 如果一个工具涉及写操作，先进入人工确认流程，不直接自动化。
- 如果一个能力无法被测试或观测，暂不进入 Agent MVP。

## 第一阶段里程碑

- M1：完成 AI Coding SOP 和 5 个可执行评估案例。
- M2：完成 Go 只读工具和 trace。
- M3：完成 C# 任务 API 基础模型。
- M4：完成 Rust 日志解析和代码索引基础函数。
- M5：用一个真实后端任务贯通主线一记录和主线二工具调用。
