# 风险与 Guardrails

## 基本原则

Agent 默认只读，写操作必须显式授权。AI Coding 默认需要人工 Review，不能直接进入生产链路。

## 必须拦截的动作

- 删除数据
- 修改生产配置
- 执行数据库写操作
- 发布部署
- 创建或删除云资源
- 向外部系统提交写请求
- 输出密钥、Token、连接串

## 输出校验

Agent 输出应尽量结构化，至少包含：

- `summary`：结果摘要
- `steps`：执行步骤
- `tool_calls`：工具调用记录
- `risks`：风险说明
- `next_actions`：建议动作

## 人工确认

当任务涉及中高风险动作时，Agent 只能返回建议和待确认计划，不能自动执行。
