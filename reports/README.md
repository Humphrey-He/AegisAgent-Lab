# Reports

本目录用于沉淀 AI Coding 工作流的阶段性复盘、评估数据和实践结论。报告应服务于后续决策：哪些任务适合 AI 参与、Prompt 如何优化、质量风险如何控制、效率收益是否稳定。

## 报告文件

- `baseline-report.md`：记录人工完成评估案例的基线耗时、质量和主要风险。
- `ai-coding-practice-report.md`：记录 AI 辅助完成评估案例的过程、结果、收益和问题。
- `agent-mvp-report.md`：预留给 Agent MVP 阶段复盘。
- `productivity-report.md`：预留给效率统计和趋势分析。
- `quality-report.md`：预留给质量缺陷、回归风险和 Review 命中率分析。
- `final-summary.md`：预留给项目总复盘。

## 记录原则

- 每条结论都应能追溯到案例、Prompt、diff、测试结果或 Review 记录。
- 不只记录成功案例，也记录失败、误报、返工和不适用场景。
- 区分人工基线、AI 辅助过程和最终人工修正后的结果。
- 记录样本数量和结论边界，避免用少量案例推出泛化结论。
- 报告中不得包含密钥、客户数据、生产敏感日志或未脱敏个人信息。
